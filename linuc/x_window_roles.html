Xウィンドウ周辺の登場人物と役割分担

登場人物を「層」で理解する — タップで詳細表示

🖥 アプリケーション Firefox / LibreOffice …
GUIアプリ本体。Xサーバに「ここにボタン描いて」と依頼する側。自分では画面を描かない。
↓ 画面描画をお願い
🪟 ウィンドウマネージャ Openbox / Mutter …
ウィンドウの「タイトルバー・枠・最小化ボタン」などの見た目を管理。ウィンドウを動かしたり重ねたりする係。
↓ 含まれる
🏠 デスクトップ環境 GNOME / KDE / Xfce …
WM＋ファイルマネージャ＋パネル＋設定ツールなどをまとめたセット。「部屋ごと」用意してくれるイメージ。
↓ 起動・ログイン管理
🔐 ディスプレイマネージャ GDM / LightDM / SDDM …
GUIのログイン画面を表示して、ユーザ認証後にデスクトップ環境を起動する係。systemdに起動してもらう。
↓ 画面出力の基盤
⚙️ Xサーバ X.Org Server …
モニタ・キーボード・マウスなどハードウェアを直接管理する基盤。アプリからの描画依頼を受けて実際に画面に出力する。
↓ 制御
🖱 ハードウェア モニタ / キーボード / マウス
物理デバイス本体。Xサーバが橋渡しして、アプリやWMが間接的に操作できるようにする。

各役割をひとことで

Xサーバ
ハードウェア（画面・マウス・キーボード）を管理する「土台」。アプリの描画依頼を受けて実際に画面へ出力する。
→ 大工さんが「基礎工事」を担当するイメージ
ウィンドウマネージャ
ウィンドウの枠・タイトルバー・見た目・重なり順を管理。Openbox Mutter など。
→ 「部屋の仕切り係」。どこに何の窓を置くか決める
デスクトップ環境
WM＋アイコン＋メニュー＋パネルなどをまとめたフルセット。GNOME KDE など。
→ 「家具付き部屋ごと」提供するイメージ
ディスプレイマネージャ
GUIログイン画面を表示し、認証後にデスクトップ環境を起動する。GDM LightDM など。systemd に起動してもらう。
→ 「受付・警備員」。入口でIDを確認して部屋に案内する

問題の選択肢を全部解剖する

①
デスクトップ環境を準備する
→ ログイン後にDEを起動するのはDMの仕事
○ 正解
②
ユーザにGUI環境のログインを提供する
→ GUIログイン画面 = DMの代表的な仕事
○ 正解
③
処理した結果をXサーバを介してウィンドウに表示させる
→ これはアプリケーションの仕事。DMは描画しない
× 不正解
④
initやsystemdによりDMサービスが起動される
→ DMはsystemdのサービスとして起動される。正しい
○ 正解
⑤
ウィンドウの概観、アイコンやカーソル、メニューなどを提供する
→ これはウィンドウマネージャ／デスクトップ環境の仕事
× 不正解
⑥
モニタの画面表示やキーボード、マウスなどのハードウェアを管理する
→ これはXサーバの仕事。DMはHWを直接触らない
× 不正解
⑦
ユーザにCUI環境のログインを提供する
→ DMはGUI専門。CUIログインはgettyやloginコマンドの仕事
× 論外

試験で混ざりやすい 3つの引っかかりポイント

⚠
DM と Xサーバを混同しやすい
どちらも「画面に関係する」イメージがあるが、Xサーバはハードウェア管理の土台、DMはログイン画面を出す受付。「ハードウェアを管理する」はXサーバの選択肢。
⚠
WM と DE を混同しやすい
WMはウィンドウの枠だけ管理。DEはWMを含むフルセット。「アイコンやメニューを提供する」はDE（またはWM）の仕事で、DMの仕事ではない。
⚠
DM は「ログイン専門」と覚える
DMの仕事は①GUIログイン画面を出す ②認証する ③DEを起動する の3つだけ。描画も、ウィンドウ管理も、HW制御もしない。
Xサーバ → 土台・ハードウェア担当
ウィンドウマネージャ → ウィンドウの見た目担当
デスクトップ環境 → WM込みのフルセット
ディスプレイマネージャ → GUIログイン受付・DE起動
DISPLAY変数 → どのXサーバに画面を送るかの指定
