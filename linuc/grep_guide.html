01 / 基本オプション
grep = 「文字列を含む行を抽出」するコマンド
オプションで「どう抽出するか」をカスタマイズする
試験頻出 — 絶対に覚えるオプション
-v
マッチした行を除外する最重要
invert（反転）の v。「含む行」ではなく「含まない行」を表示する。grep -v grep でgrepプロセス自体を除外するのが典型的な使い方。
-i
大文字・小文字を区別しない最重要
ignore case の i。grep -i error で ERROR / Error / error すべてにマッチする。
-n
行番号を表示する頻出
number の n。マッチした行の先頭に行番号を表示する。ログファイルのどこにエラーがあるか調べるときに便利。
-c
マッチした行数を表示する頻出
count の c。マッチした行の内容ではなく「何行マッチしたか」の数だけ表示する。
-l
マッチしたファイル名だけ表示する頻出
list の l。複数ファイルを検索するとき、中身ではなくファイル名だけ知りたい場合に使う。grep -l error *.log
-r
ディレクトリを再帰的に検索する頻出
recursive の r。ディレクトリ以下のファイルを全部検索する。grep -r "error" /var/log/
-E
拡張正規表現を使う
Extended regex の E。|（OR）や +（1回以上）など拡張的な正規表現が使える。egrep と同じ。
-F
正規表現を使わず固定文字列で検索
Fixed string の F。. や * を正規表現ではなくそのまま文字として扱う。fgrep と同じ。
-w
単語単位でマッチする
word の w。grep -w log は「log」という単語だけにマッチ。「login」「syslog」などは除外される。
-A n
マッチ行の後ろ n 行も表示
After の A。grep -A 3 error log.txt でエラー行とその後3行を表示。前後の文脈を確認するときに使う。
-B n
マッチ行の前 n 行も表示
Before の B。grep -B 3 error log.txt でエラー行とその前3行を表示。

03 / 派生コマンド
egrep = grep -E 拡張正規表現
Extended grep — 拡張正規表現が使える
|（OR）、+（1回以上）、?（0か1回）、{n}（n回繰り返し）などが使える。
通常の grep では \| のようにエスケープが必要なものが、そのまま書ける。
試験のポイント：「grep -E と同じ」「拡張正規表現を使う」という選択肢が来たらこれ。
egrep "error|warn" app.logerrorまたはwarnを検索
grep -E "error|warn" app.log上と同じ意味
fgrep = grep -F 固定文字列
Fixed grep — 正規表現を使わない
. や * などを正規表現として解釈せず、そのままの文字として検索する。
ピリオドやアスタリスクを含む文字列を検索するとき便利。エスケープ不要で高速。
試験のポイント：「固定文字列で検索」「正規表現を使わない」= fgrep（grep -F）。
fgrep "192.168.1.1" access.logピリオドをそのまま検索
grep -F "192.168.1.1" access.log上と同じ意味
pgrep ≠ grep オプション プロセス専用
Process grep — プロセス名でPIDを検索する専用コマンド
ps | grep の組み合わせとは別物。プロセス名で検索してPIDだけを返す。
grep プロセス自身が混入しないので、httpd問題の解決策になる。
-u で特定ユーザのプロセスのみ、-l でプロセス名も一緒に表示できる。
試験のポイント：「grepプロセスが混入する問題の解決策」「プロセス名でPIDを検索」= pgrep。
pgrep httpdhttpdのPIDだけ表示
pgrep -l httpdPID＋プロセス名を表示
pgrep -u alicealiceが実行中のプロセスPID一覧
rgrep = grep -r 再帰検索
Recursive grep — ディレクトリを再帰的に検索
指定したディレクトリ以下のファイルを全部検索する。grep -r と同じ。
試験のポイント：「ディレクトリ以下を再帰的に検索」= rgrep または grep -r。
rgrep "error" /var/log//var/log/以下を再帰的に検索
grep -r "error" /var/log/上と同じ意味
04 / 派生コマンド 比較表
コマンド同じ書き方特徴・使いどころ
egrepgrep -E拡張正規表現（| + ? {n} が使える）
fgrepgrep -F固定文字列（正規表現を使わない・高速）
rgrepgrep -rディレクトリを再帰的に検索
pgrep（別コマンド）プロセス名でPIDを検索する専用コマンド
💡
pgrep だけ別物
egrep / fgrep / rgrep は「grep のオプションに別名をつけたもの」。でも pgrep は ps コマンドの代わりになるまったく別のコマンド。混同しないように。
05 / 試験でよくある引っかけ
⚠
-v は「verbose（詳細表示）」ではなく「invert（除外）」
他のコマンドで -v は詳細表示のことが多いが、grep の -v は「マッチした行を除外する」。意味が逆なので注意。
⚠
-l は「行を表示」ではなく「ファイル名を表示」
grep -l は list（一覧）の l。行の内容ではなく、マッチしたファイルの名前だけを表示する。
⚠
grep -p は存在しない
grep -p というオプションは存在しない。ps -p（PIDで絞り込む）と混同させる引っかけ。見たら即切り。
💡
「拡張正規表現」の問題が来たら egrep と grep -E は両方正解
「全て選択」問題でどちらも選択肢にある場合は両方選ぶ。同じ意味なので両方正解になる。
