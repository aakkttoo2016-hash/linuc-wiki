01 / rwx が何を表すか
r
read（読む）
中身を見られる
cat / less / cp の元
w
write（書く）
内容を変更できる
vim / echo > / rm
x
execute（実行）
コマンドとして実行
スクリプト・バイナリ
「-」= その権限がない    rw- = 読み・書きOK、実行はNG
02 / 3ブロック構造
-|rw-|r--|r--
所有者
rw-
読み・書きOK
実行はできない
グループ
r--
読みだけ
書き・実行はNG
その他
r--
読みだけ
書き・実行はNG
03 / ls -l の出力を読む
出力例
-rw-r--r--  1  test  staff  24  Jul 21 13:38  index.html
列値意味
-rw-r--r--パーミッション先頭1文字が種別（- or d）、残り9文字が3ブロック
1ハードリンク数このファイルへのリンクの数
test所有者ファイルを作ったユーザ名
staffグループ所有グループ名（表示は1つだけ）
24サイズバイト単位（-h でKB/MB表示）
Jul 21 13:38更新日時最終更新の月日・時刻
index.htmlファイル名d から始まるパーミッションはディレクトリ

05 / chmod — パーミッションを変更する
数値モード
chmod 644 index.htmlrw-r--r-- に変更
chmod 755 deploy.shrwxr-xr-x に変更
chmod -R 755 /var/www/ディレクトリを再帰的に変更
記号モード — 対象 + 操作 + 権限
対象（WHO）
u = 所有者
g = グループ
o = その他
a = 全員
操作（OP）
+ = 追加
- = 削除
= = 上書き
権限（PERM）
r = 読み
w = 書き
x = 実行
chmod u+x script.sh所有者に実行権限を追加
chmod g-w file.txtグループから書き込みを削除
chmod o=r file.txtその他を読みのみに上書き
chmod a+r file.txt全員に読み権限を追加
06 / chown / chgrp — 所有者・グループを変更する
chown（root権限が必要）
chown alice file.txt所有者を alice に変更
chown alice:staff file.txt所有者＋グループを変更
chown :staff file.txtグループだけ変更
chown -R alice:staff /var/再帰的に変更
chgrp（グループだけ変える）
chgrp dev file.txtグループを dev に変更
chgrp -R dev /project/再帰的に変更
07 / ディレクトリの rwx はファイルと意味が違う
権限ファイルへの意味ディレクトリへの意味
r 中身（内容）を読める ls でファイル一覧を表示できる
w 内容を書き換えられる ファイルの作成・削除・名前変更ができる
x コマンドとして実行できる cd で入れる（通行証）
💡
ディレクトリの x は「通行証」
r があっても x がないと cd で入れない。x だけあると入れるが ls で一覧は見えない。
08 / 試験でよくある引っかけ
⚠
グループ名と所有者を混同しやすい
test staff 24 ... index.html → 所有者は test、グループは staff。「staffユーザが所有者」は誤り。
⚠
グループ欄の w を見落とす
rw-r--r-- のグループ欄は r--。w がないので書き込めない。「staffグループは書き込める」は誤り。
⚠
どの x 欄も見落とさない
「誰でも実行できる」という選択肢は、3ブロック全ての x 欄を縦に確認してから判断する。
💡
「全て選択」問題の解き方
各選択肢を「所有者・グループ・その他」の3ブロックで1つずつ検証する。誰のブロックが適用されるかを先に決めてから読む。
09 / 試験直前まとめ
rwx の意味
r = 読む（read）
w = 書く（write）
x = 実行（execute）
- = 権限なし
数値の対応
r=4, w=2, x=1 の合計
644 = rw-r--r--
755 = rwxr-xr-x
600 = rw-------
chmod
数値: chmod 644 file
記号: chmod u+x file
再帰: chmod -R 755 dir/
chown / chgrp
chown user file
chown user:group file
chgrp group file
※ chown は root 権限が必要
