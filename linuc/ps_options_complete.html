これだけ覚えればいい — 黄金の2パターン
ps aux
ps -ef
この2つは「全プロセスを詳しく表示」で同じ意味。どちらも正解になる。
ps aux
a = All users（全員の）
u = User name（名前も出す）
x = eXtra（端末なしも）

「全員の・名前付きで・全部」と唱える
ps -ef
-e = Everything（全部）
-f = Full format（詳しく）

「全部・詳しく」と唱える

ハイフンあり = System V流
「a だけ」「x だけ」が正解にならない理由
⚠a だけ：他ユーザのプロセスも見えるが、端末なし（バックグラウンドのデーモン等）は見えない → 全部じゃない
⚠x だけ：端末なしも見えるが、自分のプロセスだけ → 全部じゃない
✅a + x：「全ユーザ」＋「端末なしも」= 初めて全プロセスになる
研修での実物と結びつける
研修サーバで ps aux | grep nginx のように使ったことがあれば、それが「全プロセスを見てnginxだけ絞り込む」という意味だったとわかります。

ps aux → まず全員見渡す
| grep 名前 → そこから探す

この流れが体に入っていれば ps aux は永遠に忘れない。
● 絶対に覚える（試験頻出）
a
全ユーザのプロセスを表示必須
BSD流。自分以外のユーザのプロセスも見える。ただし端末ありのみ。x と組み合わせて初めて「全部」になる。
x
端末なしのプロセスも表示必須
BSD流。デーモンなど端末に紐づいていないプロセスも表示。a と組み合わせると全プロセスになる。
u
ユーザ名・CPU・メモリも表示必須
BSD流。実行ユーザ名、CPU使用率(%CPU)、メモリ使用率(%MEM)が追加表示される。ps aux の u これ。
-e
全プロセスを表示必須
System V流。a+x と同じ効果。ハイフンあり。ps -ef の e これ。
-f
フルフォーマットで表示必須
System V流。UID・PID・PPID・コマンド名などを詳細表示。BSD の f（ツリー表示）とは別物！
● 覚えておくと得（試験に出る）
f
プロセスをツリー表示頻出
BSD流（ハイフンなし）。親子関係を線で表示。-f（フォーマット）と名前が同じで紛らわしいが別物。
-l
長い形式で表示頻出
System V流。優先度(PRI・NI)やプロセス状態(S)なども表示。
-u
特定ユーザのプロセスだけ表示
System V流。ps -u root でrootのプロセスだけ。BSD の u（表示形式）と別物なので注意。
-p
特定PIDのプロセスを表示
ps -p 1234 で PID 1234 のプロセスだけ表示。
USER PID %CPU %MEM VSZ RSS TTY STAT START TIME COMMAND
root 1 0.0 0.1 16952 1876 ? Ss 09:00 0:01 /sbin/init
www-data 842 0.3 1.2 85432 12340 ? S 09:01 0:12 nginx: worker
alice 1203 0.0 0.2 21000 2048 pts/0 Ss 10:30 0:00 bash
alice 1892 0.0 0.1 10856 1024 pts/0 R+ 10:45 0:00 ps aux
USER
実行しているユーザ名
PID
プロセスID。kill するときに使う番号
%CPU / %MEM
CPU・メモリの使用率。重いプロセスを探すとき見る
TTY
端末名。? はデーモン（端末なし）
STAT
R=実行中 S=待機 D=I/O待ち Z=ゾンビ T=停止
COMMAND
実行中のコマンド名・引数
